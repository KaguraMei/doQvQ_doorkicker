# 糯叽机 Android APK 打包完整指南

## 📋 项目概述

这是一个将**糯叽机**（nuojiji）网站打包成 Android 原生 App 的项目。

### 为什么选择 Capacitor？

原网站代码中已经集成了 `window.Capacitor.Plugins` API，使用 Capacitor 打包可以：
- ✅ 保留所有原生功能（震动、通知、状态栏控制）
- ✅ 无需修改任何 JS 代码
- ✅ 自动识别并调用原生插件
- ✅ 完美支持跨域请求

如果使用 Tauri，需要手动替换所有 Capacitor API，工作量巨大且容易出错。

## 🚀 一键打包流程

### 步骤 1：同步静态文件

双击运行：
```
sync-to-www.bat
```

这个脚本会：
1. 复制 `index.html` 和 `404.html` 到 `www/` 目录
2. 复制 `assets/` 文件夹（包含所有 JS/CSS）
3. 复制图片、音频、manifest 等资源文件
4. 执行 `npx cap sync` 同步到 Android 项目

### 步骤 2：打开 Android Studio

双击运行：
```
打开AndroidStudio.bat
```

或手动执行：
```bash
npx cap open android
```

### 步骤 3：编译 APK

在 Android Studio 中：

#### 方法 A：构建 APK 文件
1. 等待 Gradle 同步完成（首次可能需要 5-10 分钟）
2. 点击菜单：`Build → Build Bundle(s) / APK(s) → Build APK(s)`
3. 等待编译完成（约 1-3 分钟）
4. 点击右下角弹出的通知 "locate" 找到 APK 文件

APK 位置：
```
android/app/build/outputs/apk/debug/app-debug.apk
```

#### 方法 B：直接运行到设备
1. 用 USB 连接 Android 手机，开启开发者模式和 USB 调试
2. 点击顶部绿色播放按钮（Run）
3. 选择你的设备
4. App 会自动安装并启动

## 📱 首次使用准备

### 1. 安装 Node.js

如果还没有安装：
- 下载：https://nodejs.org/
- 安装 LTS 版本（推荐）
- 验证：打开命令行输入 `node -v`

### 2. 安装依赖

在项目根目录打开命令行：
```bash
npm install
```

这会安装：
- Capacitor 核心库
- Android 平台支持
- 5 个原生插件（震动、浏览器、状态栏、通知、应用）

### 3. 安装 Android Studio

#### 下载安装
- 官网：https://developer.android.com/studio
- 下载 Android Studio（约 1GB）
- 运行安装程序，全部使用默认选项

#### 首次启动配置
1. 打开 Android Studio
2. 选择 "Standard" 安装类型
3. 等待下载 Android SDK（约 2-3GB）
4. 完成后可以关闭欢迎界面

## 🔄 更新流程

当原网站更新后，重新打包步骤：

### 1. 获取新的静态文件

用新版本替换项目根目录的文件：
- `index.html`
- `assets/` 文件夹
- 其他资源文件

### 2. 同步并重新编译

```bash
# 方法 1：使用脚本
双击运行：sync-to-www.bat

# 方法 2：命令行
npm run sync

# 然后打开 Android Studio 重新构建
双击运行：打开AndroidStudio.bat
```

### 3. 版本号管理

编辑 `android/app/build.gradle`：
```gradle
android {
    defaultConfig {
        versionCode 2        // 每次更新 +1
        versionName "1.1.0"  // 显示给用户的版本号
    }
}
```

## ⚙️ 高级配置

### 修改 App 名称

编辑 `capacitor.config.json`：
```json
{
  "appName": "你的应用名",
  ...
}
```

然后运行 `npx cap sync`。

### 修改包名（App ID）

**警告**：已发布的 App 不能修改包名，否则会被视为新应用。

编辑 `capacitor.config.json`：
```json
{
  "appId": "com.yourcompany.appname",
  ...
}
```

编辑 `android/app/build.gradle`：
```gradle
android {
    defaultConfig {
        applicationId "com.yourcompany.appname"
        ...
    }
}
```

### 自定义 App 图标

替换以下文件（建议使用 Android Asset Studio 生成）：
```

android/app/src/main/res/mipmap-xhdpi/ic_launcher.png
android/app/src/main/res/mipmap-xxhdpi/ic_launcher.png
android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png
```

在线生成工具：https://romannurik.github.io/AndroidAssetStudio/icons-launcher.html

## 🐛 常见问题排查

### 问题 1：白屏，什么都不显示

**检查清单**：
1. ✅ 运行过 `sync-to-www.bat` 了吗？
2. ✅ `www/` 目录下有 `index.html` 吗？
3. ✅ `www/assets/` 目录下有 JS 文件吗？
4. ✅ 运行过 `npx cap sync` 了吗？

**解决方案**：
```bash
# 重新同步
del /s /q www
sync-to-www.bat

# 或命令行
rd /s /q www
mkdir www
npm run sync
```

### 问题 2：登录失败或网络错误

**原因**：CORS 跨域限制

**检查配置**：
查看 `capacitor.config.json`：
```json
{
  "server": {
    "hostname": "nuojiji.app",
    "androidScheme": "https"
  }
}
```

**调试方法**：
1. 在 Android Studio 打开 `Logcat`（底部工具栏）
2. 筛选 `Console` 标签
3. 查看 JavaScript 错误信息
4. 搜索 "CORS" 或 "Network" 关键词

**终极方案**：
如果后端确实限制了请求来源，联系后端管理员将以下域名加入白名单：
- `https://nuojiji.app`
- `http://localhost`
- `capacitor://localhost`

### 问题 3：震动、通知等原生功能不工作

**检查插件**：
```bash
# 查看已安装的插件
npx cap ls

# 应该显示：
# @capacitor/haptics
# @capacitor/browser
# @capacitor/status-bar
# @capacitor/local-notifications
# @capacitor/app
```

**重新安装插件**：
```bash
npm install @capacitor/haptics @capacitor/browser @capacitor/status-bar @capacitor/local-notifications @capacitor/app
npx cap sync
```

**添加权限**：
编辑 `android/app/src/main/AndroidManifest.xml`，在 `<manifest>` 标签内添加：
```xml
<uses-permission android:name="android.permission.VIBRATE" />
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
<uses-permission android:name="android.permission.INTERNET" />
```

### 问题 4：Gradle 同步失败

**常见错误**：
- "Failed to download..." → 网络问题，使用代理或切换镜像源
- "SDK location not found" → Android SDK 路径未配置

**配置国内镜像**（推荐）：

创建或编辑 `android/build.gradle`：
```gradle
allprojects {
    repositories {
        maven { url 'https://maven.aliyun.com/repository/google' }
        maven { url 'https://maven.aliyun.com/repository/jcenter' }
        maven { url 'https://maven.aliyun.com/repository/public' }
        google()
        mavenCentral()
    }
}
```

### 问题 5：编译时内存不足

编辑 `android/gradle.properties`：
```properties
org.gradle.jvmargs=-Xmx2048m -XX:MaxPermSize=512m
org.gradle.daemon=true
org.gradle.parallel=true
```

## 🚢 发布到应用商店

### 1. 生成签名密钥

```bash
cd android/app
keytool -genkey -v -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
```

按提示输入密码和信息。

### 2. 配置签名

创建 `android/key.properties`：
```properties
storeFile=my-release-key.keystore
storePassword=你的密码
keyAlias=my-key-alias
keyPassword=你的密码
```

编辑 `android/app/build.gradle`：
```gradle
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    ...
    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile file(keystoreProperties['storeFile'])
            storePassword keystoreProperties['storePassword']
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}
```

### 3. 构建 Release APK

```bash
cd android
./gradlew assembleRelease
```

生成的 APK 位于：
```
android/app/build/outputs/apk/release/app-release.apk
```

### 4. 构建 AAB（Google Play）

```bash
./gradlew bundleRelease
```

生成的 AAB 位于：
```
android/app/build/outputs/bundle/release/app-release.aab
```

### 5. 上传到 Google Play Console

1. 访问：https://play.google.com/console
2. 创建应用
3. 上传 AAB 文件
4. 填写商店信息、截图等
5. 提交审核
